## R7-1 · 帳號登入（Auth / OAuth）設定評估 〔🟢 有餘力才做〕

**提出者**：R7　**對象**：R2　**優先級**：🟢 nice-to-have　**狀態**：📥 待評估

### 想做什麼
讓 Demo / 產品前端支援使用者帳號登入（例如 Google OAuth），登入後才能用對話 / 報告 /
Alert Inbox 等功能；或至少替後端 API 加一層存取保護。

### 為什麼不急
- **Demo 主軸是「能問、講得出處」**（SC-004/005/007），登入不是驗收門檻。
- 後端 thin API 目前已能端到端服務 4 場景，**無登入也能完整 demo**。
- 加 Auth 反而增加 Demo Day 斷網切備援的變數（憲法 V：要能切本地 / 預錄影片）。
- → **先列著，等核心 5 畫面 + 上雲 + 備援影片都穩了，有多餘時間再做。**

### 需要 R2 協助評估的點（待 R2 回覆）
1. **要不要做、做到哪**：Demo 階段是否真的需要登入？還是只需給 Cloud Run API 加一層保護（API key / IAM）即可？
2. **OAuth provider**：用 Google OAuth（專案已在 GCP，最順）還是其他？
3. **Auth 放哪一層**：
   - 前端（Vercel，例如 NextAuth.js）只做畫面登入，後端不驗？
   - 還是後端 Cloud Run 也要驗 token（目前 API 是否為 unauthenticated invoker？需 R2 確認部署設定）？
4. **Session / token 怎麼帶**：前端拿到 token 後如何附到 `/ask`、`/research` 請求；CORS 設定（Vercel 網域 → Cloud Run）由誰設。
5. **與憲法的關係**：登入頁 / 使用者資料是否觸及任何個資 / 金鑰存放規範（III）；不得影響 NFR-031 與斷網備援（V）。

### R7 這邊的暫定立場（未獲 R2 結論前）
- **預設不做登入**，前端維持無登入可直接用，確保 Demo 備援路徑單純。
- 若 R2 評估後認為值得加，再排進「有餘力」的時段，且需 **不破壞斷網切備援**。
