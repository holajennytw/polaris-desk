# R2 W1 D2–D5 設計：真實節點 v0（Planner / Writer）+ G1 就緒

**日期**：2026-06-02 ｜ **角色**：R2 AI 架構師·Tech Lead ｜ **狀態**：Approved（核心決策已拍板）

## 背景與限制

- 接續已合併的 `001-langgraph-skeleton`（5 節點 stub 骨架、`@traced`、固定 `ResearchState` 契約、US3 已證明換節點不必動 `workflow.py`）。
- **關鍵限制**：`.env` 內所有 API 金鑰仍是 `#` 開頭佔位字串。`settings.gemini_api_key` 實際解析為 `'# 必填…'`——**truthy 但無效**，現行 `GeminiClient.__init__` 的 `if not key` 檢查不會擋下，但真正呼叫會失敗。→ D2–D5 必須能在**無金鑰**下開發、測試、跑端到端。
- 憲法約束：成本紀律（CI 不跑 LLM）、`google-genai` SDK + `gemini-3-*-preview`、NFR-031 合規、金鑰走 `.env`/Secret Manager。

## 核心決策：Smart node + 確定性 fallback

每個升級節點都是「有真金鑰就呼叫 Gemini，否則走確定性 fallback」。

- `GeminiClient` 增 `available()`（健全的金鑰有效性判斷：空字串或 `#` 開頭 = 視為未設定），修掉 truthy-placeholder latent bug。
- 節點透過**依賴注入**收 LLM client：預設 `None` → runtime 自行決定（有金鑰建立真 client，否則 fallback）。測試注入 **fake client**，兩條路徑都可確定性驗證、CI token=0。
- 金鑰到位後**零改碼**即切真實呼叫。
