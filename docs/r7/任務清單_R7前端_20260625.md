### 🔲 1. 把所有 `catch {}` 換成 `logError`

**為什麼**：現在錯誤全部靜默，生產環境出問題看不到。

**怎麼做**：在 `frontend/src/` 搜尋 `catch {`，逐一替換：

```typescript
// 改前
} catch {
  // 靜默
}

// 改後
} catch (e) {
  logError("deleteHistory", e);
}
```

**要加 import**：
```typescript
import { logError } from "@/lib/logger";
```

**需要換的檔案**（搜尋 `catch {` 或 `catch (_`）：
- `lib/api.ts` — `deleteHistory`, `historyOne`, `chunk` 等
- `hooks/useSubscriptions.ts`
- `app/(dashboard)/notifications/page.tsx`
- `app/(dashboard)/history/page.tsx`

**預估時間**：30 分鐘

### 🔲 3. `peer/page.tsx` 的格式化函式換成 `formatters.ts`

**為什麼**：`peer/page.tsx` 有自己的 `fmtRevenue` / `fmtYoy`，現在已有集中版本，刪掉頁面版避免日後不一致。

**步驟**：
1. 在 `peer/page.tsx` 找到這些函式定義並刪除：
   ```typescript
   function fmtRevenue(...) { ... }
   function fmtYoy(...) { ... }
   ```
2. 在檔案頂部加 import：
   ```typescript
   import { fmtRevenue, fmtYoy } from "@/lib/formatters";
   ```
3. 確認呼叫點的參數型別一致（`formatters.ts` 的 `fmtRevenue` 接受千元單位）

**預估時間**：30 分鐘

### 🔲 4. 建立 mock 財務資料檔案

**為什麼**：`api.financials()` 在 mock 模式下會找 `public/mocks/financials.{ticker}.json`，檔案不存在會 404。

**步驟**：在 `frontend/public/mocks/` 建立常用 ticker 的 mock 檔：

```bash
# 需要建立的檔案（至少涵蓋 PRESETS 裡的公司）
public/mocks/financials.2330.json   # 台積電
public/mocks/financials.2317.json   # 鴻海
public/mocks/financials.2454.json   # 聯發科
```

**格式**（參考 `FinancialRow` 型別）：
```json
[
  {
    "ticker": "2330",
    "fiscal_period": "2025Q4",
    "metric_id": "revenue",
    "value": 276970000,
    "unit": "千元",
    "source_id": "mock-2330-2025Q4-revenue",
    "published_at": "2026-01-10",
    "year": 2026,
    "month": 1
  }
]
```

**預估時間**：1 小時（建 3 個檔案）
