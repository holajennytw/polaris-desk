### 16. 前端本地啟動 + 後端 debug

啟動 Next.js dev server（`npm run dev`，port 3000）+ uvicorn（port 8000），實際測試 `/research` 端點。

**發現一：「模型思考追蹤」顯示 `finish()` 答案**

`expandReActStep()` 在 `action === "finish"` 時把完整答案包成 ACT 步驟，出現在「模型思考追蹤」而非「營運重點摘要」。

**修正（`frontend/src/lib/adapters.ts`）**：

```typescript
// expandReActStep：finish 步驟不進 trace
if (step.action && step.action !== "finish") { ... }

// normalizeResearch：優先用 finish action_input（LLM 原答案）
const finishInput = raw.react_steps.slice().reverse()
  .find((s) => s.action === "finish" && s.action_input)?.action_input;
const answerText = finishInput || raw.final_answer || "";
```

**原因說明**：後端 `is_fully_traceable()` 失敗時會把 `final_answer` 換成 `_synthesize()` 降級版（格式：`- snippet（來源：source_id）`），但 LLM 真實答案只存在 `react_steps[-1].action_input`。前端應優先取 `action_input`。
