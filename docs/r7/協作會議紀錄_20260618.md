## 原始議題詳情（供日後回溯）

### 議題 1：POST/GET /history 實作責任

**衝突**：`R2_需求清單_from_R7.md §2` 暗示 R2 實作；`R3_需求清單_from_R7.md §7` 暗示 R3 實作。

**背景**：
- R2 決定儲存後端，後端 workflow 結尾自動寫入（不需前端另外呼叫）
- R2 決定儲存後端，R3 在 workflow 結尾寫入
- R2 決定儲存後端並負責端點，前端在收到 response 後另外呼叫

**決策（A/B/C）**：A=Firestore，B=R7 實作，C=前端主動呼叫

---

### 議題 2：POST /subscriptions + GET /tracking-feed 歸屬

**背景**：三個端點在 R3 清單，但功能是 CRUD + events 表查詢，不含 AI 推理；R4 已有多個讀端點。

**決策（D/E）**：GET/POST /subscriptions → R7 實作（Firestore）；GET /tracking-feed → R3；儲存後端 = Firestore
