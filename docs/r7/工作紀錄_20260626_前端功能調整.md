### 5.2 event_key 缺失診斷

BigQuery `v_financial_metrics_semantic` 有 `event_key` 欄位，但 Retriever 查詢時取不到值。

**根本原因：** `structured_store.list_financials()` SELECT 清單遺漏 `event_key`（直接原因）；`v_chunk_semantic` LEFT JOIN 命中率待驗證。

| 角度 | 結果 |
|------|------|
| SQL SELECT | ❌ `structured_store.py:84` 未 SELECT `event_key` |
| WHERE 過濾 | ✅ 沒有過濾 |
| JOIN | ⚠️ 需確認 view DDL，JOIN key 格式可能不匹配 |
| Metadata Mapping | ✅ 向量路徑已正確；結構化路徑因 SELECT 缺失而無值 |
| Embedding | ✅ `chunks` 本身無此欄，靠 `v_chunk_semantic` LEFT JOIN |
| Query Builder | ❌ `_FILTER_COLUMNS` 無 `event_key`，呼叫端無法篩選 |
| Retriever metadata | ✅ `_CITATION_METADATA_KEYS` 已含，但 JOIN 無值時被過濾為 None |

### 5.3 程式碼改動（已 commit 6eefc14 → PR #48）

#### 後端 `structured_store.py`

- `list_financials()` 新增 `granularity` 參數（`"quarter"` / `"annual"` / `"month"` / `None`）
- `annual` 邏輯：用 `year = @year` 取當年四季度列，由呼叫端加總得年度值（不用 `fiscal_period NOT LIKE '%Q%'`）
- SELECT 補 `event_key` 欄位

| granularity | WHERE 條件 | 用途 |
|------------|-----------|------|
| `"quarter"` | `month IS NULL` + `fiscal_period LIKE '%Q%'` | 單季財報 |
| `"annual"` | `month IS NULL` + `year = @year` + `fiscal_period LIKE '%Q%'` | 四季度列，呼叫端加總 |
| `"month"` | `month IS NOT NULL` | 月營收 |
| `None` | 無 | 全部（預設） |

#### 後端 `api.py`

新增 `GET /periods` 端點，前端動態期別選單來源。
