# 同業比較 — 公司選取與查詢流程說明

> 整理日期：2026-06-25  
> 適用檔案：`frontend/src/app/(dashboard)/peer/page.tsx`、`frontend/src/lib/peer.ts`  
> 對象：前端介面組員

---

## 一、問題

> 「選擇兩家公司後，在輸入框輸入問題，是否會直接抓 peer-toolbar 選擇框選的公司名，去正確抓資料？」

**答：會。** 選擇框的公司是 fallback；若問題文字沒有提到公司名稱，系統直接使用選擇框的 ticker 送出查詢。

---

## 二、程式碼流程

```
使用者在 ptb-slot-A / ptb-slot-B 點選公司
  → setAId(id) / setBId(id)                        ← 儲存在 React state

使用者在輸入框打問題 → Enter 或送出按鈕
  → runQuery(問題文字)

    parseQuery(問題文字)
      → 嘗試從文字裡辨識公司名稱別名（COMPANY_ALIASES）
      → 回傳 ordered[]（辨識到的公司清單）、period、tab

    nextAId = ordered[0]?.id  ??  aId   ← 文字沒辨識到 → 用選擇框
    nextBId = ordered[1]?.id  ??  bId   ← 文字沒辨識到 → 用選擇框

    if (!nextAId || !nextBId) return     ← 兩者都沒有 → 不送出

  → runQueryWith(query, nextAId, nextBId, fiscalPeriod, ...)
      → POST /peer-compare { a_ticker, b_ticker, fiscal_period, question }
```

關鍵程式碼（`peer/page.tsx` L570–571）：

```typescript
const nextAId = ok[0]?.id ?? aId;  // 文字找不到公司 A → 用選擇框
const nextBId = ok[1]?.id ?? bId;  // 文字找不到公司 B → 用選擇框
```

## 三、三種使用情境

| 情境 | 選擇框 | 問題文字 | 結果 |
|---|---|---|---|
| **常規使用** | A=台積電、B=聯發科 | 「毛利率怎麼比？」（無公司名） | ✅ 用選擇框兩家公司查詢 |
| **文字覆蓋** | A=台積電、B=聯發科 | 「鴻海 vs 聯詠 法說重點」 | ✅ 文字公司覆蓋選擇框，並更新選擇框顯示 |
| **都沒有** | 空 | 「毛利率怎麼比？」（無公司名） | ❌ `nextAId` / `nextBId` 為空，直接 return，不送查詢 |

## 四、重要限制：文字辨識只認識 4 家公司

`parseQuery`（`lib/peer.ts` `COMPANY_ALIASES`）目前只能從文字辨識以下公司：

| 輸入別名 | 對應 ticker |
|---|---|
| 台積電、tsmc、TSMC、台積 | 2330 |
| 鴻海、foxconn、Foxconn | 2317 |
| 聯發科、mediatek、MediaTek | 2454 |
| 聯詠、novatek、Novatek | 3034 |

**其他公司**（例如國泰金、廣達、聯電）在問題文字裡打名字**不會被辨識**，必須透過選擇框選取。若未選取且文字也辨識不到，查詢不會送出。

### 完整解法（待 R3 實作）

後端 `POST /resolve` 語意識別端點（`R3_需求清單_from_R7.md §10`）：  
能從「半導體龍頭」、「國泰金控」等自然語言解析出 ticker，取代目前前端有限的 alias 字典。

## 五、`readyToCompare` 與 UI 狀態

```typescript
const readyToCompare = aId && bId;  // 兩個選擇框都有選
```

| 狀態 | 「重新分析」按鈕 | 提示文字 |
|---|---|---|
| 兩個選擇框都空 | 停用 | 「請從上方選擇，或是輸入問題後開始分析」 |
| 兩個選擇框都有選 | 啟用 | 「公司已選定，送出查詢後顯示比較結果」 |
| 查詢進行中 | 停用（loading） | — |

> 注意：Enter 送出走 `runQuery()`，有自己的 `if (!nextAId || !nextBId) return` 保護，與 `readyToCompare` 獨立。

## 六、期別（ptb-period）與公司選取的關係

期別選單的選項來源：

```typescript
// 優先使用 BQ 真實資料（載入後）
availablePeriods = [...aFinRows, ...bFinRows]
  .map(r => r.fiscal_period).unique().sort().reverse()

// BQ 資料未載入時 fallback 靜態清單
|| PERIOD_OPTIONS  // ["2026Q2","2026Q1","2025Q4",...]
```

當選擇框換公司時，`useFinancials(aId)` 和 `useFinancials(bId)` 重新取值，`availablePeriods` 自動更新。若目前選的期別不在新清單內，`useEffect` 自動切換到最新期別。
