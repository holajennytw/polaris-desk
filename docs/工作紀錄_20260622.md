## 重要發現：`POST /subscriptions` 失敗根因

**症狀**：訂閱設定公司清單顯示正常，但點擊切換訂閱顯示「儲存失敗」。

**根因**：Cloud Run 後端缺少 `GOOGLE_CLIENT_ID` 環境變數。

- `src/polaris/config.py:77`：`google_client_id: str = ""`（預設空字串）
- `src/polaris/auth.py:verify_token()`：呼叫 `verify_oauth2_token(..., audience=settings.google_client_id)`
- `audience=""` → 任何真實 token 的 `aud` 都不符 → 丟例外 → 回 `None`（匿名）
- `_require_uid(None)` → HTTP 401 `{"detail": "需要登入"}`

**不是 R3 的問題**：後端程式碼正確，只缺部署設定。

**修法（R1/R2 執行）**：
```bash
gcloud run services update polaris-api \
  --region asia-east1 \
  --set-env-vars GOOGLE_CLIENT_ID=<OAuth 2.0 Client ID>
```

**前端 token 傳法正確**：NextAuth `jwt` callback 已把 `account.id_token` 存進 session；`authHeaders()` 送 `Authorization: Bearer`；CORS preflight 回 200，允許 localhost:3000 + authorization header。

## 監控系統警示「固定顯示」根因分析（2026-06-22 下午）

### 症狀
研究助理與同業比較的「監控系統警示」面板，在兩頁之間切換時會互相顯示對方的警示，且查詢前就有內容。

### 根因

**問題一（已修）：`contraAlertStore` 跨頁共用**

- `contraAlertStore` 原本用單一 `sessionStorage` key `polaris_contra_alerts`
- 研究助理跑完矛盾偵測 → 寫入 store → 切換到同業比較頁 → `useContraAlerts()` 讀到相同 key → 警示出現在錯誤頁面
- **修法**：改為 page-scoped key（`polaris_contra_alerts_research` / `polaris_contra_alerts_peer`）；`contraAlertStore.get/set/clear/subscribe` 全部加 `page: "research" | "peer"` 參數；`useContraAlerts` 接受 page 參數

**問題二（待 R3）：`GET /alerts` 無 `origin` 欄位**

- 後端 `GET /alerts` 回傳所有 watchdog 警示，無法區分屬於哪個頁面
- 目前兩頁都直接吃全量 `alerts`，代碼中有 TODO 注解等 R3 補欄位
- **R3 需求**：`GET /alerts` 回傳的每筆 alert 加 `origin: "research" | "peer" | "global"` 欄位
- **前端修法**（R3 交付後）：
  ```ts
  // research/page.tsx
  const researchAlerts = [...(alerts ?? []).filter(a => !a.origin || a.origin === "research"), ...contraAlerts];
  // peer/page.tsx
  const peerAlerts = [...(alerts ?? []).filter(a => !a.origin || a.origin === "peer"), ...contraAlerts];
  ```
